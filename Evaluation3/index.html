Client and Server Communication

The client is the user device(browser) requesting information. The server responds and to requests and delivers content.

Web development focuses on creating both sides for a sealess experience!!

The Journey of a Web Request

When you enter a URL in the browser, here's what happens:

  1. Browser looks up the domain name via DNS.
  2. Browser sends a HTTP request to the server.
  3. Server processes the request and sends back a response.
  4. Browser renders the webpage for the user.
